# シードスクリプトの実行方法

## 現在の状況

Prisma 7の仕様変更により、シードスクリプトの直接実行でエラーが発生しています。

## 解決方法

### 方法1: Prisma Studioを使用して手動でデータを投入（推奨）

1. Prisma Studioを起動：
   ```bash
   cd date-guide
   npm run db:studio
   ```

2. ブラウザで `http://localhost:5555` にアクセス

3. 各テーブルにデータを手動で追加：
   - `purpose_categories` - 用途カテゴリ（初デート、誕生日、記念日など）
   - `restaurants` - 店舗データ
   - `restaurant_purposes` - 店舗と用途の関連

### 方法2: SQLを直接実行

SupabaseのSQL Editorで以下のSQLを実行：

```sql
-- 用途カテゴリの作成
INSERT INTO purpose_categories (id, name, description, is_active, created_at, updated_at)
VALUES
  (gen_random_uuid(), '初デート', '初めてのデートにおすすめのお店', true, NOW(), NOW()),
  (gen_random_uuid(), '誕生日', '誕生日のデートにおすすめのお店', true, NOW(), NOW()),
  (gen_random_uuid(), '記念日', '記念日のデートにおすすめのお店', true, NOW(), NOW()),
  (gen_random_uuid(), 'カジュアルデート', '気軽に楽しめるデートにおすすめのお店', true, NOW(), NOW()),
  (gen_random_uuid(), '夜のデート', '夜のデートにおすすめのお店', true, NOW(), NOW())
ON CONFLICT (name) DO NOTHING;
```

### 方法3: シードスクリプトの修正（今後の課題）

Prisma 7の仕様に合わせて、シードスクリプトを修正する必要があります。
