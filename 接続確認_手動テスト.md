# データベース接続の手動テスト方法

接続URLが正しいか確認するために、以下の方法でテストできます。

## 方法1: SupabaseのSQL Editorで確認

1. Supabaseダッシュボード → `date-guide` プロジェクトを選択
2. 左メニューから **「SQL Editor」** をクリック
3. 以下のSQLを実行：

```sql
SELECT version();
```

これが実行できれば、データベースは正常に動作しています。

## 方法2: psqlコマンドで接続テスト（Mac/Linux）

もし、`psql` がインストールされている場合：

```bash
psql "postgresql://postgres:dateguide202512@db.alnqctmakkhlzwezfibj.supabase.co:5432/postgres"
```

接続できれば、プロンプトが表示されます。接続できない場合は、エラーメッセージが表示されます。

## 方法3: Prisma Studioで確認

```bash
cd premier-date
npm run db:studio
```

これでPrisma Studioが起動し、データベースに接続を試みます。

---

## 次のステップ

接続URLを再取得して、`.env` ファイルを更新したら、再度マイグレーションを実行してください：

```bash
npm run db:migrate
```
