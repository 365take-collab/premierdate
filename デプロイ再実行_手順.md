## 🚀 デプロイを再実行

環境変数を追加した後、デプロイを再実行して変更を反映します。

### 方法A: Vercelダッシュボードから再デプロイ（簡単・推奨）

#### ステップ1: Deploymentsタブを開く

1. **Vercelダッシュボード**: https://vercel.com/dashboard
2. **premier-date** プロジェクトを選択
3. **Deployments** タブをクリック

#### ステップ2: 最新のデプロイメントを選択

1. 最新のデプロイメントを確認
2. デプロイメントの右側にある **...** メニューをクリック
3. **Redeploy** を選択

#### ステップ3: デプロイの確認

1. デプロイが開始されることを確認
2. デプロイが完了するまで待つ（通常数分）
3. デプロイが成功したことを確認

---

### 方法B: GitHubから自動デプロイ（推奨）

GitHubにプッシュすると、Vercelが自動的にデプロイを実行します。

#### ステップ1: 変更をコミット

```bash
cd /Users/kawamuratakeshi/Cursor/premier-date
git add .
git commit -m "ドメイン設定を追加"
```

#### ステップ2: GitHubにプッシュ

```bash
git push
```

#### ステップ3: 自動デプロイの確認

1. **Vercelダッシュボード** → **Deployments** タブを開く
2. 新しいデプロイメントが開始されることを確認
3. デプロイが完了するまで待つ（通常数分）

## ✅ デプロイ完了後の確認

### 1. デプロイの成功確認

1. **Deployments** タブで、最新のデプロイメントの状態を確認
2. ✅ **Ready** または ✅ **Success** になっているか確認

### 2. 環境変数の反映確認

デプロイが完了したら、環境変数が正しく反映されているか確認：

1. **Deployments** → 最新のデプロイメントをクリック
2. **Build Logs** または **Runtime Logs** を確認
3. 環境変数が読み込まれているか確認

### 3. サイトの動作確認

1. **Deployments** → 最新のデプロイメントの **Visit** ボタンをクリック
2. サイトが正常に表示されるか確認
3. メール認証リンクなど、環境変数を使用する機能が正常に動作するか確認

## 🔍 トラブルシューティング

### 問題1: デプロイが失敗する

**原因**: ビルドエラーや環境変数の設定ミス

**解決策**:
1. **Deployments** → 失敗したデプロイメントをクリック
2. **Build Logs** を確認してエラーを特定
3. エラーを修正して再度デプロイ

### 問題2: 環境変数が反映されない

**原因**: デプロイが再実行されていない、または環境変数がProduction環境に設定されていない

**解決策**:
1. 環境変数の設定画面で、各環境変数が **Production** にチェックされているか確認
2. デプロイを再実行
3. デプロイログで環境変数が読み込まれているか確認

### 問題3: メールリンクがlocalhostになっている

**原因**: 環境変数が正しく設定されていない、またはデプロイが再実行されていない

**解決策**:
1. Vercelの環境変数を確認（`NEXTAUTH_URL`が`https://premierdate.jp`になっているか）
2. Production環境に設定されているか確認
3. デプロイを再実行
